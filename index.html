Cloud Computing Lab Viva Questions and Answers

01. Hosted Virtualization using VirtualBox and KVM

Hosted virtualization runs virtual machines (VMs) on top of an existing operating system (OS) using virtualization software like VirtualBox or VMware Workstation.
Example: You install VirtualBox on your Windows laptop and run Ubuntu Linux inside it as a virtual machine.

VirtualBox is a free and open-source virtualization software that lets you create and run multiple operating systems on a single physical computer.
Example: A developer can test a website in both Windows and Linux using VirtualBox without needing two computers.

KVM (Kernel-based Virtual Machine) is a virtualization tool built into Linux. It turns a Linux OS into a hypervisor, allowing you to run virtual machines directly on your system.
Example: On a Linux server, KVM lets you host multiple VMs like a web server, database server, etc., all on the same machine.

02. Bare-Metal Virtualization using Xen, Hyper-V, VMware ESXi

It is a type of virtualization where the hypervisor is installed directly on the physical hardware, without any operating system underneath.
Example: VMware ESXi installed directly on a server to run multiple VMs for different departments in a company.

Hyper-V is Microsoft's hypervisor and integrates well with Windows Server. VMware ESXi is a popular hypervisor from VMware, used widely in enterprise environments.
Example: If a company uses mostly Windows servers, they might choose Hyper-V. For cross-platform flexibility, VMware ESXi is better.

Xen is an open-source Type 1 hypervisor that allows multiple OSes to run on the same hardware.
Example: Amazon EC2 used Xen for virtualization before moving to its new Nitro system.

03. Infrastructure as a Service (IaaS) using AWS

Infrastructure as a Service provides virtualized computing resources over the internet like servers, storage, and networking.
Example: AWS EC2 lets you launch virtual servers that you can configure with your own software.

Amazon EC2 (Elastic Compute Cloud) gives you virtual servers to manage just like a physical machine.
Example: A company can host a website by launching an EC2 instance and installing a web server like Apache on it.

04. Platform as a Service (PaaS) using AWS Elastic Beanstalk

Platform as a Service offers a complete development and deployment environment in the cloud.
Example: Deploying a Node.js app without managing the server or database manually.

AWS Elastic Beanstalk helps developers deploy and manage applications easily. Just upload your code and AWS does the rest.
Example: A developer can upload a Python app and Elastic Beanstalk will automatically deploy it with the required environment.

05. Identity and Access Management (IAM) in AWS

IAM is a service that helps you manage users, groups, roles, and permissions securely in AWS.
Example: Giving one user access to only S3 while another can manage EC2 instances.

IAM protects your cloud resources from unauthorized access. It ensures users only get the access they need.
Example: In a company, the HR team gets access to employee data but not to servers or billing.

06. Security as a Service on AWS/Azure

It is a cloud-based security solution that includes firewalls, intrusion detection, and encryption.
Example: AWS WAF (Web Application Firewall) protects your web apps from attacks like SQL injection.

  • AWS Shield – protects against DDoS attacks
  • Azure Security Center – security recommendations and threat detection
  • AWS GuardDuty – detects threats and anomalies

07. Storage as a Service using AWS S3, Glacier, Azure

It allows you to store, manage, and access your data in the cloud instead of on hard drives.
Example: You upload files to S3 and access them from anywhere.

S3 is used for frequently accessed data. Glacier is for long-term backup and infrequent access.
Example: Store your project files in S3 and old college notes in Glacier.

08. Database as a Service using AWS RDS, Azure SQL

A fully managed cloud database service. No need to manage setup, updates, or backups.
Example: AWS RDS lets you launch a MySQL database with just a few clicks.

AWS RDS helps you set up and manage relational databases like MySQL, PostgreSQL, and SQL Server.
Example: An e-commerce site can use RDS to store customer and order data.

SQL uses tables with fixed schemas. NoSQL has flexible schemas and supports documents, key-values, graphs, etc.
Example: Use SQL for banking data; NoSQL like MongoDB for social media feeds.

09. Containerization using Docker on AWS/Azure/GCP

Packaging software and its dependencies into a container that runs the same anywhere.
Example: A container can hold a Python app and its libraries, ensuring it runs the same everywhere.

Docker is a tool to create, deploy, and manage containers.
Example: Developers use Docker to ship apps without worrying about the environment differences.

Containers are lightweight, portable, fast, and ensure consistency across environments.
Example: A startup uses Docker to deploy their app on both AWS and Azure without changing code.
